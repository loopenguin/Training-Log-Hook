# 훈련일지 자동화 시스템 안내서

> 이 문서는 기술 담당자가 아닌 운영·행정 담당자를 위해 작성되었습니다.
> 시스템이 **어떻게 작동하는지**, **무엇을 확인해야 하는지**를 중심으로 설명합니다.

---

## 이 시스템은 무엇을 하나요?

매일 오후 5시 30분, 사람이 직접 하던 **훈련일지 작성 작업을 자동으로 대신 수행**합니다.

기존에는 담당자가 직접:
1. LMS(Develrocket)에 접속하여 출결 현황 확인
2. 구글 출결 시트와 대조하여 지각·결석·공가 등 분류
3. 훈련일지 페이지에 해당 내용 직접 입력·저장

이 과정을 수동으로 반복했다면, 이제는 시스템이 이를 **전부 자동으로** 처리합니다.

---

## 작동 흐름 (7단계)

```
① 자동 시작
    ↓
② LMS 로그인
    ↓
③ 오늘 날짜의 훈련일지 페이지 이동
    ↓
④ '출석정보 불러오기' 버튼 자동 클릭 → 출결 데이터 수집
    ↓
⑤ 구글 출결 시트에서 공가·외출 등 출석 정보 확인
    ↓
⑥ 훈련일지 '비고' 란에 결과 자동 입력 후 저장
    ↓
⑦ 디스코드로 완료 보고서 전송
```

## 디스코드 보고서 읽는 법

작업 완료 후 디스코드에 아래와 같은 형태의 메시지가 전송됩니다.

```
[SUCCESS] 26년 4월 9일 목요일 훈련일지 자동화 실행 완료
실행 시각(Asia/Seoul): 2026. 4. 9. PM 5:30:12

[확인 필요 대상]
홍길동
김영희

[타겟 사이트 기입 로그]
사이트 직접 제출 성공

┌──────────────────────────────┐
  1. 지각 (1명)
  - 조광조 (09:35)

  2. 결석 (2명)
  - 홍길동 (*확인 필요*)
  - 박문수 (*확인 필요*)

  3. 공가-출석인정 (3명)
  - 강감찬 (휴가)
  - 구지용 (병가)
  - 도경수 (예비군)

  4. 외출 (1명)
  - 김영삼 (*확인 필요*)

  5. 조퇴 (0명)
  -
└──────────────────────────────┘
```

### 각 항목 설명

| 항목 | 의미 |
|---|---|
| `[SUCCESS]` | 모든 작업이 정상 완료됨 |
| `[확인 필요 대상]` | 시트에 사유가 기입되지 않아 담당자 직접 확인이 필요한 인원 |
| `[타겟 사이트 기입 로그]` | LMS 훈련일지에 실제로 저장이 됐는지 여부 |
| `(*확인 필요*)` | 구글 시트에 해당 인원의 상세 사유가 없음. **수동 보완 필요** |

## ⚠️ 담당자가 해야 할 일

시스템이 모든 것을 대신하지는 않습니다. **아래 상황은 담당자가 직접 처리**해야 합니다.

### 1. `(*확인 필요*)` 인원 처리
디스코드 메시지에서 `[확인 필요 대상]`에 이름이 나온 경우:
- 해당 훈련생의 결석·외출 사유를 운영매니저 또는 당사자에게 직접 확인
- LMS 훈련일지 페이지에 접속하여 해당 항목을 수동으로 수정

### 2. 구글 출결 시트 관리
시스템은 **구글 출결 시트에 기입된 내용을 기준**으로 공가·외출 여부를 판단합니다. <br>
시트에 누락이 있으면 자동화 결과도 부정확해질 수 있으므로, **시트는 항상 최신 상태**로 유지해 주세요.

### 3. 공휴일 목록 관리
임시공휴일 지정 등으로 달력이 바뀌면, 담당 기술자에게 요청하여 공휴일 목록을 갱신해야 합니다.

### 4. 외부 크론 PAT 만료 시 조치
cron-job.org가 GitHub API를 호출할 때 사용하는 **GitHub PAT에 만료일이 있습니다.** 토큰 만료 시 자동화가 중단됩니다.
- 만료일 전후: GitHub 설정에서 새 토큰 발급 후, cron-job.org 잡 설정의 `Authorization` 헤더 값만 교체
- 담당 기술자 연락: PAT 만료로 시스템이 오전 실행되지 않아 디스코드 메시지가 오지 않으면 가장 먼저 확인 필요
- 세부 교체 절차는 `docs/ENV_SETUP.md · 섹션 5.4` 참조

## ❌ 실패 메시지가 왔을 때

```
[FAILURE] 훈련일지 자동화 실행 실패
실패 시각(Asia/Seoul): 2026. 4. 9. PM 5:31:05
실패 단계: SITE_CRAWL
오류 메시지: ...
```

| 확인 사항 | 조치 |
|---|---|
| `SITE_CRAWL` 실패 | LMS 사이트 접속 불가 또는 로그인 정보 만료. IT 담당자에게 전달 |
| `DATA_TRANSFORM` 실패 | 출결 시트 구조가 바뀌었거나 오늘 날짜 컬럼 부재. IT 담당자에게 전달 |
| `SITE_SUBMIT` 실패 | LMS 훈련일지 저장 실패. IT 담당자에게 전달 |
| 디스코드 메시지 자체가 안 올 때 | GitHub Actions 장애 또는 웹훅 URL 만료. IT 담당자에게 전달 |

> 실패 시에도 디스코드로 실패 사유가 전송되므로 채널을 주기적으로 확인해 주세요.

## 자주 묻는 질문

**Q. 수동으로 훈련일지를 먼저 작성하면 어떻게 되나요?**
시스템은 기존에 입력된 내용을 덮어쓰므로, **자동 실행 전에 작성한 내용은 사라질 수 있습니다.** 수동 작성이 필요한 경우 IT 담당자에게 해당 일자의 자동화를 비활성화 요청하세요.

**Q. 오늘 훈련이 없는 날(휴강 등)은 어떻게 되나요?**
공휴일로 미리 등록된 날만 건너뜁니다. 등록되지 않은 휴강일에는 시스템이 평소대로 실행되므로 **담당 기술자에게 사전 공지**해야 합니다.

**Q. 기입 내용을 나중에 수정할 수 있나요?**
네. LMS 훈련일지 페이지에서 직접 수정 가능합니다. 자동화 결과는 '초안' 개념으로 사용하고, 확인 필요 항목은 수동으로 보완하는 방식을 권장합니다.
