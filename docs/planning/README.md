# 앱 기획서 가이드

> 프로젝트 시작 전, 아래 흐름에 따라 기획서를 완성하세요.
> claude-code가 기획서를 바탕으로 태스크를 분해하고 에이전트에 위임합니다.

## 사용법

1. **앱 유형 선택** → 해당 유형의 기획 템플릿 열기
2. **각 항목 작성** → 가이드를 참고하여 빈칸 채우기
3. **완성된 기획서를 `docs/planning/spec.md`로 저장**
4. claude-code에게 "기획서 기반으로 작업 시작해" 요청

## 공통 항목 (모든 유형)

아래 항목은 유형에 관계없이 반드시 정의해야 합니다.

### 1. 프로젝트 개요

| 항목 | 가이드 | 예시 |
|------|--------|------|
| **프로젝트명** | 작업영역 식별에 사용. 영문 kebab-case 권장 | `curriculum-bot` |
| **한 줄 설명** | 이 앱이 무엇인지 한 문장으로 | "교안 자동 생성 도우미" |
| **앱 유형** | 위 표에서 선택 | 웹 앱 |
| **기술 스택** | 사용할 언어/프레임워크. 미정이면 "자동 선택" | HTML/JS/CSS (바닐라) |

### 2. 사용자 정의

| 항목 | 가이드 |
|------|--------|
| **주 사용자** | 이 앱을 쓸 사람은 누구인가 (역할, 기술 수준) |
| **사용 환경** | 데스크톱/모바일, 브라우저 종류, 오프라인 필요 여부 |
| **핵심 시나리오** | 사용자가 앱을 열고 → 무엇을 하고 → 어떤 결과를 얻는가 (3~5단계) |

### 3. 디자인 참조

| 항목 | 가이드 |
|------|--------|
| **디자인 파일 경로** | 외부 도구(Google Stitch, Figma 등)에서 생성한 design.md 경로. claude-code가 Read로 읽어 스펙에 반영 |
| **스크린샷/목업 경로** | 이미지 파일 경로. gemini-cli에 위임하여 분석 |
| **디자인 시스템** | 사용 중인 디자인 토큰, 컴포넌트 라이브러리 (있으면) |

> 디자인 파일이 없으면 이 섹션을 "없음"으로 표시. UI 워크플로우의 Phase 1(gemini-cli 디자인 리서치)에서 대체합니다.

### 4. 범위 정의

| 항목 | 가이드 |
|------|--------|
| **MVP 범위** | 최소한 동작해야 하는 기능 목록 (1차 릴리스) |
| **향후 확장** | 나중에 추가할 수 있는 기능 (지금은 구현 안 함) |
| **명시적 제외** | 이 앱이 하지 않는 것 (오해 방지) |

## 기획서 품질 체크리스트

기획서 완성 후 아래를 확인하세요:

- [ ] 모든 필수 항목이 채워져 있는가
- [ ] "자동 선택" / "미정"인 항목이 3개 이하인가
- [ ] 핵심 시나리오가 구체적인 동사로 서술되어 있는가
- [ ] MVP 범위와 향후 확장이 명확히 구분되어 있는가
- [ ] 모호한 표현("적절히", "좋은 UX") 대신 구체적 기준이 있는가
