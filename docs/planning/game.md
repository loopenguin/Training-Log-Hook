## 2. 시스템 설계

> 가이드: 게임의 규칙과 메커니즘을 정의합니다.
> 구현 방법이 아닌 "게임 규칙"을 서술하세요.

### 2-1. 코어 루프

```
[플레이어 행동] → [시스템 반응] → [피드백] → [다음 행동 유발]
예: 블록 배치 → 줄 완성 판정 → 점수+이펙트 → 난이도 상승
```

### 2-2. 게임 메커니즘

| 메커니즘 | 설명 | 관련 수치/변수 |
|---------|------|---------------|
| **이동/조작** | 플레이어가 할 수 있는 기본 행동 | |
| **승리 조건** | 게임을 클리어하는 조건 | |
| **패배 조건** | 게임 오버 조건 (없으면 "없음") | |
| **점수 체계** | 점수 산정 방식 (없으면 "없음") | |
| **난이도 곡선** | 시간/진행에 따른 난이도 변화 | |

### 2-3. 상태 관리

```
게임 상태:
  - phase: 타이틀 / 플레이 / 일시정지 / 결과
  - score: number
  - level: number
  - (게임별 고유 상태)
```

## 4. 플로우차트

> 가이드: 게임의 전체 화면 흐름과 상태 전이를 정의합니다.

### 4-1. 화면 흐름

```
[타이틀]
   │
   ├─ 게임 시작 → [게임 플레이]
   │                  │
   │                  ├─ 일시정지 → [일시정지 메뉴]
   │                  │                 ├─ 계속 → [게임 플레이]
   │                  │                 └─ 나가기 → [타이틀]
   │                  │
   │                  ├─ 클리어 → [결과 화면]
   │                  │              └─ 다시/다음 → [게임 플레이]
   │                  │              └─ 타이틀로 → [타이틀]
   │                  │
   │                  └─ 게임 오버 → [결과 화면]
   │
   ├─ 설정 → [설정 화면] → [타이틀]
   └─ (도움말 등)
```

### 4-2. 인게임 턴/프레임 흐름

```
매 프레임/턴:
  1. 입력 수집 (키보드/터치/마우스)
  2. 게임 로직 업데이트
  3. 충돌/조건 판정
  4. 상태 갱신 (점수, HP, 위치 등)
  5. 렌더링
```
